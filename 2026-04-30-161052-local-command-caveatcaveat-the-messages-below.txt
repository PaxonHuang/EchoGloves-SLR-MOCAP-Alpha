в—Ҹ Design Section 2: RingBuffer (Sliding Window)

  New file: glove_firmware/lib/Filters/RingBuffer.h

  A fixed-size circular buffer for SensorData frames, implementing the SOP's
  sliding window requirement:
  - Capacity: WINDOW_SIZE (30 frames = 300ms at 100Hz)
  - Type: SensorData frames stored contiguously in a SensorData[30] array
  - Push: O(1) вҖ” write to head position, advance head = (head + 1) % 30
  - IsFull: returns true once 30 frames have been pushed
  - CopyOut: copies all 30 frames in chronological order to a flat
  float[WINDOW_SIZE * FEATURE_COUNT] buffer for model input, applying
  normalization inline
  - Reset: clears buffer and resets head/tail

  The ring buffer is owned by Task_SensorRead (single-writer) and read by
  Task_Inference (when isFull() is signaled via task notification). No mutex
  needed since the SOP specifies single-writer/single-reader with
  notification-based synchronization.

  Memory: 30 Г— sizeof(SensorData) вүҲ 30 Г— ~180 bytes = ~5.4KB. Fits in ESP32-S3
  SRAM easily.
