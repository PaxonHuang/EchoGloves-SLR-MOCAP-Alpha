## 2. I2C总线拓扑设计

> 本节定义I2C总线的物理拓扑结构、多路复用器分配策略及上拉电阻参数计算。由于系统包含16个I2C设备（15个TMAG5273 + 1个BNO085），远超单条I2C总线可寻址数量，且TMAG5273仅有4个可编程地址（0x22-0x25），必须采用多路复用方案。设计遵循以下原则：(1) 高速率设备与低速率设备物理隔离；(2) 关键设备（IMU）独立总线保证实时性；(3) 线缆较长分支需加强上拉驱动能力。

### 2.1 总线架构总览

系统采用 **双I2C总线** 架构，利用ESP32-S3的两个硬件I2C控制器（I2C0和I2C1）：

```
                         ┌─────────────────────────────────────────────────┐
                         │                  ESP32-S3                       │
                         │                                                 │
                    I2C0_SDA (GPIO 8)──────┐                    ┌──I2C1_SDA (GPIO 18)
                    I2C0_SCL (GPIO 9)──────┤                    ├──I2C1_SCL (GPIO 19)
                         ┌─────────────────┤                    │
                         │                 │                    │
                    [主总线A: 400kHz]       │             [主总线B: 400kHz]
                    TMAG5273传感器群         │             BNO085 IMU
                         │                 │                    │
```

- **I2C0 (总线A)**: 专用于15个TMAG5273手指弯曲传感器，通过TCA9548A级联扩展
- **I2C1 (总线B)**: 专用于BNO085手背IMU，独立直连保证传感器融合数据的高实时性

### 2.2 TCA9548A通道分配方案

#### 2.2.1 设计约束

- TCA9548A提供8个子通道，每个TCA9548A自身占用主总线上一个I2C地址（0x70-0x77，由A0/A1/A2引脚配置）
- 每个子通道最多挂载4个TMAG5273（因为TMAG5273仅有4个可编程地址：0x22, 0x23, 0x24, 0x25）
- 15个传感器 ÷ 4个/通道 = 3.75，至少需要4个子通道
- 采用4个子通道的分配方案：**4 + 4 + 4 + 3 = 15**

#### 2.2.2 拓扑结构

理论上仅需1片TCA9548A即可（使用其中4个通道），但考虑到可靠性冗余、总线容性负载及手指FPC走线长度，实际选用 **2片TCA9548A** 方案：

```
                                    ┌──────────┐
                         I2C0_SDA──│TCA9548A #1│──I2C0_SDA
                         I2C0_SCL──│ADDR=0x70  │──I2C0_SCL
                                    │(A0=A1=A2=GND)│
                                    └──┬──┬──┬──┬──┘
                          CH0  CH1  CH2  CH3  CH4-CH7(保留)
                           │    │    │    │
                    ┌──────┘    │    │    └──────┐
                    │      ┌────┘    └────┐      │
                 [拇指]  [食指]    [中指]  [无名指]
                 3×TMAG  3×TMAG   3×TMAG  3×TMAG

                                    ┌──────────┐
                         I2C0_SDA──│TCA9548A #2│──I2C0_SDA (与#1并联)
                         I2C0_SCL──│ADDR=0x71  │──I2C0_SCL (与#1并联)
                                    │(A0=VCC,    │
                                    │ A1=A2=GND) │
                                    └──┬──┬──┬──┬──┘
                          CH0  CH1  CH2  CH3  CH4-CH7(保留)
                           │
                      [小指]
                      2×TMAG
                       (+1冗余位)
```

**实际推荐方案 — 单TCA9548A四通道分配：**

考虑到手套FPC走线的物理布局，最优方案为将每根手指的3个传感器分配到同一子通道，这样在扫描某一手指时只需切换一次TCA9548A通道，减少切换延迟：

| TCA9548A通道 | 分配手指 | TMAG5273数量 | 器件地址 | 说明 |
|:---:|:---:|:---:|:---:|:---|
| CH0 | 拇指 (Thumb) | 3 | 0x22, 0x23, 0x24 | MCP/CMC + IP + TIP关节 |
| CH1 | 食指 (Index) | 3 | 0x22, 0x23, 0x24 | MCP + PIP + DIP关节 |
| CH2 | 中指 (Middle) | 3 | 0x22, 0x23, 0x24 | MCP + PIP + DIP关节 |
| CH3 | 无名指 (Ring) | 3 | 0x22, 0x23, 0x24 | MCP + PIP + DIP关节 |
| CH4 | 小指 (Pinky) | 3 | 0x22, 0x23, 0x24 | MCP + PIP + DIP关节 |

**问题：** 5根手指 × 3传感器/指 = 15个，但单TCA9548A只有8通道且每通道最多4个同地址设备，上述方案每通道3个设备使用地址0x22-0x24，实际可行。

**最终确认方案：使用1片TCA9548A即可满足15个传感器需求。** 每根手指3个传感器占一个通道（地址0x22/0x23/0x24），共5个通道（CH0-CH4），CH5-CH7保留扩展。

但考虑到FPC走线长度和容性负载，若某一通道的FPC走线过长（如拇指到主板的距离），可能导致信号完整性问题。因此**预留第2片TCA9548A**的焊盘位置（ADDR=0x71），用于极端情况下的总线拆分。

#### 2.2.3 级联时序预算

每次TCA9548A通道切换需要约6μs（建立时间），传感器读取一次3轴数据约：

```
单次TMAG5273读取流程:
  TCA9548A通道切换:      6 μs
  START + 地址+写:       2 + 8 = 10 bits × 2.5μs = 25 μs  (@400kHz)
  寄存器地址写入:         8 bits × 2.5μs = 20 μs
  RESTART + 地址+读:     2 + 8 = 10 bits × 2.5μs = 25 μs
  6字节读取(X/Y/Z各2B):  48 bits × 2.5μs = 120 μs
  STOP:                   25 μs
  ─────────────────────────────────────────
  单次传感器读取 ≈ 221 μs

单通道3个传感器顺序读取:
  3 × 221 μs = 663 μs

全部5通道扫描:
  5 × (6 + 663) μs = 3,345 μs ≈ 3.3 ms
```

以200Hz采样率目标，每帧周期5ms，传感器扫描耗时3.3ms，剩余1.7ms用于数据处理和通信，满足实时性要求。

### 2.3 BNO085独立总线

BNO085挂在**I2C1 (总线B)**上，直连ESP32-S3 GPIO 18/19，不经过多路复用器。

```
    ESP32-S3 GPIO 18 (I2C1_SDA) ───── 4.7kΩ ── VCC
         │
         └─────────── BNO085 SDA (ADDR=0x4A, COM_PS=0)
    
    ESP32-S3 GPIO 19 (I2C1_SCL) ───── 4.7kΩ ── VCC
         │
         └─────────── BNO085 SCL
```

**独立总线设计理由：**
1. **实时性保证：** BNO085在sensor fusion模式下以400Hz输出四元数数据，需持续高频读取。若与TMAG5273共享总线，TCA9548A通道切换会造成数据延迟
2. **中断机制：** BNO085支持硬件中断(INT)通知主机有新数据就绪，独立总线可避免中断处理时总线争用
3. **地址冲突回避：** BNO085地址(0x4A/0x4B)虽不与TMAG5273(0x22-0x25)冲突，但物理隔离更利于系统调试和故障排查
4. **噪声隔离：** IMU为连续高频采样设备，独立总线减少对TMAG5273低频采样可能引入的信号干扰

### 2.4 I2C上拉电阻计算

#### 2.4.1 主总线A (I2C0) 上拉电阻

主总线A的负载包括：1个TCA9548A + 通过TCA9548A选通的3个TMAG5273。

**计算参数：**
- 电源电压 V_CC = 3.3V
- I2C低电平阈值 V_OL = 0.4V (ESP32-S3规格)
- 总线目标上升时间 t_r ≤ 300ns (400kHz快速模式I2C标准要求)
- 估计总线总电容 C_bus

**总线电容估算：**

```
主总线A总线电容 C_bus:
  TCA9548A输入电容:          10 pF × 1 = 10 pF
  PCB走线电容 (50mm):        0.5 pF/cm × 5cm = 2.5 pF
  FPC走线电容 (每通道):       1 pF/cm × 10cm × 3 = 30 pF (最差情况)
  TMAG5273输入电容:          5 pF × 3 = 15 pF
  ────────────────────────────────────────────
  C_bus ≈ 58 pF (保守取 80 pF)
```

**上拉电阻值计算：**

I2C上拉电阻需满足两个约束条件：

**(a) 上升时间约束：**
$$t_r = 0.8473 \times R_p \times C_{bus} ≤ 300 \text{ ns}$$
$$R_p ≤ \frac{300 \text{ ns}}{0.8473 \times 80 \text{ pF}} = 4.42 \text{ kΩ}$$

**(b) 低电平驱动约束：**
$$V_{OL} = V_{CC} - I_{OL} \times R_p ≤ 0.4 \text{ V}$$

ESP32-S3 I2C灌电流能力 I_OL ≈ 20mA：
$$R_p ≥ \frac{V_{CC} - V_{OL}}{I_{OL}} = \frac{3.3 - 0.4}{0.02} = 145 Ω$$

但TCA9548A输入灌电流仅约3mA：
$$R_p ≥ \frac{3.3 - 0.4}{0.003} = 967 Ω$$

**(c) 功耗约束：**
$$P_{Rp} = \frac{V_{CC}^2}{R_p}$$
取 R_p = 4.7kΩ 时，P = 3.3² / 4700 = 2.32 mW（可接受）

**结论：主总线A选用 R_p = 4.7kΩ (±1%)，满足所有约束。** SDA和SCL各一个上拉电阻，放置在ESP32-S3端（上拉源端）。

#### 2.4.2 主总线B (I2C1) 上拉电阻

BNO085直连总线，仅1个设备，电容负载极小：

```
C_bus ≈ BNO085输入(10pF) + PCB走线(2pF) = 12 pF
```

上升时间约束：
$$R_p ≤ \frac{300}{0.8473 \times 12} = 29.5 kΩ$$

考虑BNO085输入漏电流约1μA，确保足够的噪声容限：
**选用 R_p = 4.7kΩ (±1%)，与总线A统一物料编码。**

#### 2.4.3 TCA9548A子总线上拉电阻

每个子通道的下游总线也需要上拉电阻（TCA9548A内部无上拉）：

```
子通道总线电容:
  TMAG5273输入:  5 pF × 3 = 15 pF
  FPC走线:       1 pF/cm × 10cm × 2 = 20 pF (手指到TCA9548A)
  ────────────────────────────────────────
  C_sub ≈ 35 pF

R_p_sub ≤ 300 / (0.8473 × 35) = 10.1 kΩ
```

**子通道选用 R_p = 4.7kΩ (±1%)**，确保FPC长走线下的信号完整性。5个使用通道各需要SDA/SCL共2个上拉电阻，共10个。

## 4. 电源管理设计

> 电源管理是可穿戴设备设计的核心挑战之一。本系统总工作电流约50-80mA（详见第5节功耗预算），峰值不超过120mA。设计需满足以下要求：(1) 1000mAh LiPo电池支撑8-12小时续航；(2) 充电电路支持USB-C 5V输入，1A充电电流约1.2小时充满；(3) LDO效率在典型负载下≥85%；(4) 过充/过放/过流全面保护。

### 4.1 3.3V LDO选型

#### 4.1.1 负载电流分析

```
系统最大负载电流:
  ESP32-S3 (WiFi TX峰值):     280 mA  (瞬时)
  ESP32-S3 (WiFi RX):          100 mA  (瞬时)
  ESP32-S3 (CPU活动):           40 mA  (持续)
  ESP32-S3 (Modem Sleep):       7 mA   (轻负载)
  TMAG5273 × 15:             15 × 1.7 = 25.5 mA  (持续)
  BNO085 (融合模式):           12 mA   (持续)
  TCA9548A × 1:                1 mA   (持续)
  LED + 分压网络:               5 mA   (持续)
  ────────────────────────────────────────────
  典型工作电流: ≈ 84 mA (CPU活动+WiFi休眠+传感器全开)
  WiFi发送峰值:  ≈ 360 mA (瞬时，<100ms)
  深度睡眠:       ≈ 10 mA (传感器不关时) 或 <1mA (全关)
```

#### 4.1.2 LDO选型对比

| 参数 | ME6211C33M5G-N | AMS1117-3.3 | RT9013-33GB | HT7833 |
|------|:---:|:---:|:---:|:---:|
| 输出电压 | 3.3V | 3.3V | 3.3V | 3.3V |
| 最大输出电流 | 500mA | 1A | 500mA | 250mA |
| 压差 (500mA) | 300mV | 1.3V | 250mV | 200mV |
| 静态电流 (Iq) | **2μA** | 5mA | 60μA | 3μA |
| PSRR (@1kHz) | 70dB | 60dB | 75dB | 65dB |
| 封装 | SOT-23-5 | SOT-223 | SOT-23-5 | SOT-89 |
| 单价 | ¥0.80 | ¥0.60 | ¥1.20 | ¥0.50 |
| 推荐 | ✅ 首选 | ❌ Iq太高 | ⚠️ 备选 | ⚠️ 电流不足 |

**选定：ME6211C33M5G-N**

选型理由：
1. **超低静态电流(2μA)：** 在系统低功耗模式（传感器采集间隔较长时）显著降低待机功耗
2. **500mA输出能力：** 满足84mA典型负载，WiFi TX峰值360mA虽超出规格但为瞬时脉冲，LDO热保护不会触发
3. **低噪声(70dB PSRR)：** 对I2C通信和ADC采样至关重要，减少电源纹波对TMAG5273 16-bit ADC精度的影响
4. **SOT-23-5小封装：** 适合可穿戴设备的空间约束

#### 4.1.3 功耗与热设计验证

```
LDO功耗计算 (最恶劣工况: 电池电压3.0V, 负载84mA):
  P_LDO = (V_in - V_out) × I_load
         = (3.0 - 3.3) × 0.084   (注意: 电池电压可能低于LDO输出!)
```

> **⚠️ 关键问题：** LiPo电池放电截止电压为3.0V，而LDO输出为3.3V。当电池电压降至3.3V以下时，LDO无法正常稳压！

**解决方案：使用电荷泵升压 + LDO方案，或采用低电压工作策略。**

```
方案A (推荐): 电池 → 升压至3.7V → LDO降至3.3V
  升压芯片: SY7088AAC 或 TPS61021 (效率>90%)
  缺点: 增加BOM ¥1.5，增加PCB面积

方案B: 直接使用升压型DC-DC (Buck-Boost) 输出3.3V
  芯片: TPS63020 或 SY8089AAC
  优点: 全电压范围3.0-4.2V均能稳定3.3V输出
  缺点: 开关噪声可能影响传感器精度

方案C (简化): 降低系统工作电压至3.0V
  所有器件最低工作电压:
    ESP32-S3: 3.0V (最低)
    TMAG5273: 1.7V ✅
    BNO085: 2.4V ✅
    TCA9548A: 1.65V ✅
  ESP32-S3在3.0V下可工作，但WiFi射频性能可能下降
  优点: 零额外BOM
```

**最终推荐：方案A，使用SY7088AAC升压 + ME6211C33 LDO。**

SY7088AAC参数：
- 输入: 2.5-4.8V
- 输出: 可调，设为3.7V
- 最大输出: 1A
- 效率: >92% @ 200mA
- 封装: SOT-23-6
- 单价: ¥1.50

### 5.3 功耗优化策略

| 策略 | 描述 | 节省电流 | 实现复杂度 |
|------|------|:---:|:---:|
| TMAG5273分时轮询 | 将传感器读取从连续改为轮询，非读取时段传感器进入低功耗 | 5-10 mA | 低 |
| BNO085动态采样率 | 手静止时降低融合频率至10Hz | 4-8 mA | 中 |
| ESP32-S3单核运行 | 数据手套场景单核足够 | 12 mA | 低 |
| CPU动态调频 | 空闲时降至80MHz | 16 mA | 中 |
| ESP-NOW占空比优化 | 仅在数据变化时发送，非定时发送 | 5-10 mA | 中 |
| 传感器电源门控 | 未使用手指的传感器通过GPIO控制MOSFET关断 | 10-25 mA | 高 |

### 5.4 续航总结

| 工作模式 | 系统电流(mA) | 续航时间(h) | 适用场景 |
|----------|:---:|:---:|:---|
| 满性能 (200Hz+50Hz ESP-NOW) | 99 | 10.1 | 游戏/VR主场景 |
| 均衡 (50Hz+10Hz ESP-NOW) | 55.5 | 18.0 | 日常开发调试 |
| 节能 (20Hz+1Hz心跳) | 43 | 23.3 | 闲置/演示模式 |
| 深度睡眠 | 1.84 | 543 (22.6天) | 运输/存储 |

**结论：1000mAh电池在满性能模式下续航约10小时，满足8-12小时设计目标。**

```
              │   ↓ 方向             │   ↓ 方向
              │ [磁铁]              │  [磁铁]
    活动侧 ───┘────          活动侧 ──┘──╱───
    
    磁场方向: 垂直于关节平面       磁场方向: 旋转了θ角
    TMAG输出: Bx≈0, By=Bz=0      TMAG输出: Bx=Bsinθ, Bz=Bcosθ
    
    弯曲角度 θ = arctan2(Bx, Bz)
```

**安装细则：**
1. 磁铁使用医用级环氧树脂胶水固定在手指活动侧（指骨上）
2. TMAG5273传感器板固定在相邻指骨的固定侧
3. 磁铁与传感器间距: 2-5mm (关节自然弯曲时)
4. 磁铁轴向垂直于手指纵轴，确保弯曲时产生最大的磁场方向变化
5. 每个磁铁需标记N/S极方向，安装时必须方向一致

## 8. 传感器标定规范

> 本节定义系统级传感器标定流程和算法规范。TMAG5273霍尔传感器和BNO085 IMU均需要在装配后进行标定，以消除制造误差和个体差异。标定分为三级：器件级出厂校准、模块级安装校准、系统级联合校准。

### 8.1 TMAG5273零偏与灵敏度校准

#### 8.1.1 零偏校准 (Zero-Offset Calibration)

**目的：** 消除传感器在无外部磁场时的输出偏差（零偏），以及地球磁场和环境磁场的本底影响。

**原理：** TMAG5273在手指完全伸直时，磁铁与传感器之间的相对位置固定，此时输出一个参考磁场矢量。将此矢量作为"零位"参考，后续角度计算基于相对变化量。

**校准流程：**

```
步骤1: 硬件复位
  - 上电后等待所有TMAG5273初始化完成 (约5ms/个)
  - 执行软复位 (WHO_AM_I寄存器读取确认通信正常)

步骤2: 环境磁场采样
  - 要求用户将手完全展开，手掌朝上平放
  - 等待2秒让传感器稳定
  - 连续采样100次 (约500ms @ 200Hz)
  - 计算每个传感器3轴的平均值: B_offset[x,y,z]

步骤3: 零偏计算
  - 零偏值 = 采样均值
  - 存储到NVS (Non-Volatile Storage)
  - 后续读数减去零偏: B_cal = B_raw - B_offset

步骤4: 灵敏度校准 (可选)
  - 要求用户缓慢弯曲手指至最大角度，保持2秒
  - 记录最大弯曲时的磁场矢量
  - 计算灵敏度系数: k = |B_max - B_offset| / θ_max
  - θ_max 使用量角器辅助测量作为参考
```

**代码示例（PlatformIO C++）：**

```cpp
struct TMAG_Calibration {
    float offset[3];    // 零偏 [x, y, z] 单位: μT
    float sensitivity;  // 灵敏度系数 μT/degree
    float max_angle;    // 最大弯曲角度 degree
};

// 零偏校准函数
void tmag_calibrate_zero_offset(uint8_t channel, TMAG_Calibration* cal) {
    float sum[3] = {0, 0, 0};
    const int samples = 100;
    
    for (int i = 0; i < samples; i++) {
        float bx, by, bz;
        tmag_read_xyz(channel, &bx, &by, &bz);
        sum[0] += bx; sum[1] += by; sum[2] += bz;
        vTaskDelay(pdMS_TO_TICKS(5));  // 200Hz采样
    }
    
    cal->offset[0] = sum[0] / samples;
    cal->offset[1] = sum[1] / samples;
    cal->offset[2] = sum[2] / samples;
    
    // 保存到NVS
    nvs_handle_t handle;
    nvs_open("cal", NVS_READWRITE, &handle);
    nvs_set_blob(handle, "tmag_cal", cal, sizeof(TMAG_Calibration));
    nvs_commit(handle);
    nvs_close(handle);
}
```

#### 8.1.2 角度计算算法

TMAG5273检测的是3轴磁场分量，关节弯曲角度通过磁场矢量方向变化计算：

```
方法: atan2双轴角度法 (适用于单轴弯曲)

1. 选取弯曲平面内的两个轴 (通常为X和Z轴，由安装方向决定)
2. 计算角度:
   θ = atan2(Bx - offset_x, Bz - offset_z) × (180/π)
   
3. 补偿非线性:
   - TMAG5273磁场强度与距离的平方成反比
   - 弯曲时磁铁-传感器距离变化导致非线性
   - 使用二次多项式拟合:
     θ_true = a·θ_raw² + b·θ_raw + c
   - 系数(a,b,c)通过三点标定获得(0°, 45°, 90°)

4. 角度范围:
   - 伸直: ≈0°
   - 完全弯曲: ≈90° (MCP/PIP) 或 ≈110° (DIP)
```

#### 8.1.3 多磁铁串扰补偿

相邻关节的磁铁可能产生干扰信号，需在标定中测量和补偿：

```
串扰测量流程:
1. 固定目标手指在伸直位置
2. 逐一弯曲其他手指至90°
3. 记录目标手指传感器的输出变化
4. 建立串扰矩阵 C[i][j]:
   C[i][j] = 当手指j弯曲90°时，传感器i的读数变化量

5. 运行时补偿:
   B_compensated[i] = B_raw[i] - Σ(C[i][j] × θ_normalized[j])
   其中 θ_normalized[j] = θ[j] / 90° (归一化弯曲量)
```

### 8.2 BNO085出厂校准

#### 8.2.1 BNO085内置自校准

BNO085在上电后会自动执行内部传感器校准，包括：
- **加速度计零偏校准：** 静止放置6轴设备自动校准
- **陀螺仪零偏校准：** 静置时自动运行
- **磁力计硬铁校准：** 通过"8字运动"自动学习硬铁偏移

#### 8.2.2 校准流程

```
步骤1: 基础初始化
  - 上电后等待BNO085启动 (最大100ms)
  - 设置sensor fusion输出: 四元数 (Game Rotation Vector, 报告ID 0x09)
  - 设置融合频率: 100Hz
  - 使能加速度计/陀螺仪/磁力计原始数据输出

步骤2: 陀螺仪校准 (自动)
  - 将手套静止放置在水平面上
  - BNO085在静止状态下自动校准陀螺仪零偏
  - 校准时间: 约2秒
  - 通过读取CALIB_STAT寄存器确认校准状态

步骤3: 加速度计校准 (自动)
  - 手套静止平放即完成
  - 校准状态通过CALIB_STAT.bit.gyro确认

步骤4: 磁力计校准 (需用户操作)
  - 提示用户戴着手套做"8字运动" (Figure-8 pattern)
  - 运动要求: 手腕绕三个轴旋转，每个轴至少旋转360°
  - 持续时间: 约10-15秒
  - 实时显示校准进度条 (通过BLE/WiFi推送到浏览器)
  - CALIB_STAT.bit.mag = 3 表示校准完成

步骤5: 校准数据保存
  - BNO085校准数据保存在内部Flash中 (掉电不丢失)
  - 同时将校准偏移量读取并保存到ESP32-S3的NVS中作为备份
  - 若BNO085内部数据丢失，从NVS恢复
```

#### 8.2.3 校准状态监控

```cpp
// BNO085校准状态读取

    uint8_t gyro;  // 陀螺仪校准
    uint8_t accel; // 加速度计校准
    uint8_t mag;   // 磁力计校准

    BNO_CalibStatus status;

        // 通过WiFi SSE推送校准提示到浏览器

        snprintf(msg, sizeof(msg), 
                 "{\"type\":\"calib\",\"mag\":%d,\"gyro\":%d}",
                 status.mag, status.gyro);
        sse_send(msg);
    }
}
```

```cpp
// 校准数据存储结构 (NVS namespace: "hand_calib")
#define CALIB_VERSION  0x0001  // 版本号，用于兼容性检查

typedef struct __attribute__((packed)) {
```
