## rf.ino 最终方案代码工作流

````cpp
setup() {
    初始化MPU6050传感器
    初始化5个柔性传感器
    校准传感器范围
}

loop() {
    // 1. 采集传感器数据
    读取5个柔性传感器值 → x[0:5]
    读取陀螺仪数据 → x[5:8]
    读取加速度计数据 → x[8:11]
    
    // 2. 数据归一化
    flex值 / (maxInput - minInput)
    加速度 / 9.81
    
    // 3. 本地推理
    randomForest.predict(x) → 预测类别
    
    // 4. 输出结果
    Serial打印预测标签 (A/B/J/W/Y)
}
````

## 详细讲解：Random Forest在ESP32上的推理分类算法

### 一、系统整体流程

```
传感器数据采集 → 特征提取与归一化 → 多决策树投票 → 取多数投票 → 输出分类结果
```

---

### 二、数据采集与特征提取

#### 2.1 传感器配置
系统使用**11维特征向量**：

```cpp
x[0-4]:   5个柔性传感器数据（flex sensors）
x[5-7]:   陀螺仪数据（gyroscope）  - x, y, z 三轴
x[8-10]:  加速度计数据（accelerometer）- x, y, z 三轴
```

#### 2.2 数据归一化公式

对于柔性传感器：
$$x_i = \frac{\text{sensorValue}_i - \text{minInput}_i}{\text{maxInput}_i - \text{minInput}_i}$$

其中：
- `sensorValue`：实时采集的原始值
- `minInput` / `maxInput`：传感器的标定最小值和最大值
- 目的：将不同传感器的数据缩放到 $[0, 1]$ 范围内

加速度数据归一化：
$$\text{ACC}_{\text{normalized}} = \frac{\text{ACC}_{\text{raw}}}{9.81}$$

代码示例：
```cpp
// 柔性传感器归一化
if (maxInput > minInput) {
    x[i] = (sensorValue - minInput) / (maxInput);
} else {
    x[i] = 0;
}

// 陀螺仪和加速度数据直接分配
x[5] = g.gyro.x;      // 陀螺仪x轴
x[8] = a.acceleration.x / 9.81;  // 加速度x轴
```

### 三、Random Forest推理核心算法

#### 3.1 算法原理

Random Forest（随机森林）是**集成学习**算法，通过多个决策树的民主投票进行分类：

$$\text{Prediction} = \text{argmax}\left(\sum_{t=1}^{T} \text{Vote}_{tree_t}\right)$$

其中：
- $T = 30$：决策树总数（从classifier.h看有tree #1到tree #30）
- 每棵树对每个类别进行投票
- 最终选择获得最多票数的类别

#### 3.2 训练参数（Python）

从RF.py可以看到训练配置：

```python
model = RandomForestClassifier(
    n_estimators=2000,      # 训练时使用2000棵树
    max_depth=5,            # 每棵树最大深度为5
    min_samples_split=4,    # 节点分裂最少样本数4个
    min_samples_leaf=5,     # 叶子节点最少样本数5个
    max_leaf_nodes=150,     # 最多150个叶子节点
    max_samples=50,         # 每棵树使用50个样本
    random_state=15,
    oob_score=True          # 启用袋外（OOB）验证
)
```

训练后通过模型编译器转换为**C++代码**（classifier.h），在ESP32上以原始决策树逻辑运行。

### 四、决策树分裂逻辑详解

#### 4.1 单棵树的推理过程

以Tree #1为例：

```cpp
// tree #1
if (x[3] <= -0.22238226979970932) {
    if (x[0] <= 0.14451148733496666) {
        votes[2] += 1;  // 类别2（J手势）投票
    }
    else {
        votes[4] += 1;  // 类别4（Y手势）投票
    }
}
else {
    if (x[2] <= 1.122367024421692) {
        votes[1] += 1;  // 类别1（B手势）投票
    }
    else {
        votes[3] += 1;  // 类别3（W手势）投票
    }
}
```

**分裂流程**：
1. 先检查 `x[3]`（陀螺仪y轴）是否 $\leq -0.222$
   - 若是，继续检查 `x[0]`（flex sensor 1）是否 $\leq 0.144$
     - 是 → 投票给类别2
     - 否 → 投票给类别4
   - 若否，继续检查 `x[2]`（flex sensor 3）是否 $\leq 1.122$
     - 是 → 投票给类别1
     - 否 → 投票给类别3

#### 4.2 决策树的数学表示

决策树的分裂可以表示为分段常数函数：

$$f_t(x) = \sum_{l=1}^{L_t} c_{t,l} \cdot \mathbb{1}(x \in R_{t,l})$$

其中：
- $L_t$：第t棵树的叶子节点数
- $R_{t,l}$：第l个叶子节点对应的特征空间区域
- $c_{t,l}$：该区域对应的投票类别
- $\mathbb{1}(·)$：示函数（if-else判断）

### 五、投票与最终分类

#### 5.1 投票累计机制

```cpp
uint8_t votes[5] = { 0 };  // 5个类别的投票计数器
// tree #1, #2, ..., #30 都会对votes[0-4]进行累计投票
```

#### 5.2 取多数投票（Majority Voting）

```cpp
uint8_t classIdx = 0;
float maxVotes = votes[0];

for (uint8_t i = 1; i < 5; i++) {
    if (votes[i] > maxVotes) {
        classIdx = i;           // 更新最大投票类别
        maxVotes = votes[i];    // 更新最大票数
    }
}
return classIdx;
```

**公式**：
$$\hat{y} = \arg\max_{k \in \{0,1,2,3,4\}} \sum_{t=1}^{30} \mathbb{1}(\text{tree}_t \text{ votes for class } k)$$

### 九、核心公式总结

**特征归一化**：
$$x_i = \frac{v_i - v_{min}}{v_{max} - v_{min}}$$

**单棵树推理**：
$$\text{Vote}_{tree}(x) \in \{0, 1, 2, 3, 4\}$$

**集成投票**：
$$\text{Prediction} = \arg\max_c \sum_{t=1}^{30} \mathbb{1}(\text{Vote}_{tree_t}(x) = c)$$

**置信度**（可选）：
$$\text{Confidence} = \frac{\max(\text{votes})}{\sum_c \text{votes}_c} = \frac{\max(\text{votes})}{30}$$

---

这就是Random Forest在ESP32上推理分类的完整详解！核心思想是**多棵弱决策树通过民主投票形成强分类器**，既保证了准确率，又满足嵌入式设备的资源限制。
